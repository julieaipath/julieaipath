你是不是每周/月都有个固定工作，就是给甲方做项目进度汇报，那我们看看这个问题怎么用Claude cowork来解决。让我们乙方通过指挥AI工作也能体验一把做甲方的感受。

# 以前怎么做？

把零散的工作记录汇总 → 自己整理到word → 手动调整格式 → 发给甲方。

一个甲方走完这套流程大概 1 小时出头。多个甲方走完，一个下午就没了。关键是这事儿特别机械，没什么脑力含量，纯粹是重复劳动。每次做完都有种"我今天啥正事没干啊"的虚无感。

## 用Claude cowork怎么做？

我用的工具是 Claude Cowork 的桌面端。

我在桌面上建了一个文件夹，叫"工作进展汇报"。里面放了两个文档：`20260509_张三项目进展.docx` 和 `20260508_李四项目进展.docx`。就是平时我自己记的那些零碎工作记录，格式很随意，有些是随手打的列表，有些是几句流水账。我这里举例用的编的数据，你就按照自己的实际工作内容来。

然后我对 Cowork 说了一句话：

"帮我把这个目录下的 2 份文件，分别整理成本周工作进展汇报。输出成两个独立的 HTML 文件，排版整齐，让客户一眼看清楚我这周做了什么。"

然后它会读那两个文档。在我的文件夹里生成了两个 HTML 文件，我双击打开——排版工整，分好了"项目整体概况"、"本周完成工作"等几个区块，并且排版非常的美观。

我当时的表情大概就是：🤯

从我说完那句话，到两个文件摆在桌面上，前后不到3分钟。

## 真正让我拍大腿的是下一步

如果你觉得上面那个操作只是"帮我省了整理和排版的时间"，后面更牛的是，我让它把刚才那个流程存成了一个"技能（Skill）"。过程很简单——就是把刚才那个指令打包，给它起了个名字叫"/generate\-work\-report "。以后再想做这件事，我不用再次输入那段话，直接输入这个指令"/generate\-work\-report "就行。

这还没完。

我进一步把它设成了一个定时任务。每周五上午8点，Cowork 自动读取我那个文件夹里本周新增的工作记录文件，自动生成排版好的周报 HTML，放在指定目录下。

对，你没看错。它自己到了点就干活，根本不用我管。

那一刻的感觉很奇妙，有种AI助理更智能了的感觉。

## 说说它哪里好，和哪里还不够好

用了段时间，我想坦白说说感受。

好的方面很明显：它真的能读懂你零碎的工作记录。我之前担心它会把各个项目的进展搞混，毕竟两个文档放在同一个目录下。但它的表现很稳定，每个文档的内容区分得很清楚，生成的周报各自独立。

还有就是排版审美在线。默认的 HTML 模板看着挺专业，有颜色分区，有进度条样式，甲方打开的时候体验感不错。不用我再额外调整格式。

但说说不那么满意的地方。

它不会自动"发现"新文件。你想生成汇报时，需要提前把相关资料放到目录下。不过话说回来，这本来就是正常的，和cowork的设计有关，你在当前目录下工作，它只能获取当前目录的操作权限，其他目录的权限它没有。而且AI 帮你干活的前提是你得给它素材。它不可能获取你聊天信息中的内容。如果能获取到那也太可怕了，人就没有一点隐私了。
